01 — Propósito

Introducción y Objetivos Técnicos

"Desarrollar una aplicación Android con Kotlin/Java que actúe como cliente principal del sistema FestivalTrack, consumiendo una API REST para la gestión de agenda cultural, venta de boletos y streaming en vivo del Festival José Alfredo Jiménez."

El sistema se estructura como una aplicación cliente-servidor: la app Android consume una API REST backend desplegada en Railway, con PostgreSQL como base de datos principal y Redis para gestión de sesiones y caché. El streaming en vivo se entrega mediante protocolo HLS adaptativo a través de ExoPlayer.

El proyecto es el componente web/móvil de un ecosistema multiplataforma más amplio que contempla también dispositivos wearable y Smart TV.

Requerimientos No Funcionales

📈

Escalabilidad

Mínimo 1,000 usuarios concurrentes durante transmisiones en vivo sin degradación del servicio.

⏱️

Disponibilidad

Uptime 24/7 con reconexión automática ante pérdida de señal. SLA objetivo: 99.5%.

🔐

Seguridad

Autenticación JWT, HTTPS obligatorio, pasarela de pago cifrada y permisos declarados en AndroidManifest.xml.

🔧

Mantenibilidad

Arquitectura MVVM, cobertura de pruebas unitarias mínima del 80%, código documentado con KDoc.

📱

Compatibilidad

Android API Level 26+ (Android 8.0). Diseño responsivo adaptado a diferentes tamaños de pantalla.

⚡

Rendimiento

Latencia WebSocket < 1 segundo. Stream HLS inicia en menos de 5 segundos. Carga de pantallas < 3 segundos.

03 — Estructura

Vista Lógica — Arquitectura MVVM

El proyecto implementa el patrón MVVM (Model-View-ViewModel), estándar recomendado por Google para aplicaciones Android modernas. La separación de capas permite testear la lógica de negocio de forma independiente a la UI.
